# FitnessProgressor – Service Layer

This repository contains the backend service layer for the FitnessProgressor application.  
It includes all API endpoints required for the MVP, along with the database design and service logic.

## Purpose
The service layer is responsible for:
- Handling API requests from the UI
- Managing workout, user, insights, and recommendation data
- Communicating with the PostgreSQL database
- Returning structured JSON responses

## Repository Structure
- `/design` – Contains database design and service layer design documents
- `/src` – Will contain backend implementation files in future sprints

## Status
This repository is currently in Sprint 1 setup. Development begins in Sprint 2.
